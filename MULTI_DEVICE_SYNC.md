# Руководство: Работа на нескольких компьютерах (Mac / Windows)

Этот документ описывает, как правильно синхронизировать работу над проектом между несколькими устройствами (например, вашим Mac и Windows-ПК) с использованием вашего форка на GitHub.

## 1. Первичная настройка на новом компьютере (Windows)

Чтобы начать работу на новом устройстве, вам нужно склонировать ваш форк:

```bash
git clone https://github.com/Elguajo/SurfSense.git
cd SurfSense
```

## 2. Что синхронизируется, а что НЕТ

**Синхронизируется (переносится через GitHub):**
* Весь исходный код проекта.
* Ваши изменения и новые файлы (если они не добавлены в `.gitignore`).
* История коммитов.

**НЕ синхронизируется (остается только локально):**
* **Файлы с настройками, паролями и ключами (`.env`):** В целях безопасности они игнорируются. Вам нужно будет вручную скопировать файл `.env` (и другие `.env` файлы, например в `surfsense_backend/.env`) с вашего первого компьютера (Mac) или создать их заново по примерам.
* **Базы данных и кэш (`docker/data/`):** Данные PostgreSQL, Redis и т.д. не отправляются на GitHub, так как весят много и зависят от платформы. При первом запуске Docker на новом компьютере (Windows) будут созданы чистые, пустые базы данных.
* **Скрытые локальные папки (`.kilo/` и др.):** Локальные настройки и логи не засоряют репозиторий.

## 3. Повседневный рабочий процесс (Workflow)

Чтобы изменения не потерялись и не возникало конфликтов, придерживайтесь следующего цикла при смене устройства:

### На Компьютере 1 (например, Mac):
1. Поработали и внесли изменения.
2. Сохранили изменения локально в Git:
   ```bash
   git add .
   git commit -m "Краткое описание того, что сделали"
   ```
3. Отправили в свой форк на GitHub:
   ```bash
   git push origin main
   ```

### На Компьютере 2 (например, Windows):
1. **ОБЯЗАТЕЛЬНО** перед началом работы скачайте последние изменения:
   ```bash
   git pull origin main
   ```
2. Поработали и внесли новые изменения.
3. Сохранили и отправили их на сервер:
   ```bash
   git add .
   git commit -m "Описание изменений с Windows"
   git push origin main
   ```

### Возврат на Компьютер 1 (Mac):
1. Опять же, обязательно скачайте обновления перед тем, как писать код:
   ```bash
   git pull origin main
   ```
2. И так далее по кругу.

## 4. Обновление из оригинального проекта (Upstream)

Когда в оригинальном репозитории (у оригинального автора) появляются новые классные фичи или исправления багов, вы можете подтянуть их в свой проект, не потеряв свои собственные изменения.

**Шаг 1. Настройка связи с оригиналом** *(нужно сделать один раз на каждом компьютере)*:
```bash
git remote add upstream https://github.com/MODSetter/SurfSense.git
```

**Шаг 2. Получение обновлений** *(когда автор что-то обновил)*:
```bash
# Скачиваем информацию о новых коммитах
git fetch upstream

# Вливаем новые коммиты в наш локальный код
git merge upstream/main

# Если всё прошло без конфликтов, отправляем объединенный код в свой GitHub форк
git push origin main
```
